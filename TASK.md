# Тестовое задание: Финансовое API

Реализовать простое финансовое API-приложение.

### Технологии

- Ruby + Ruby on Rails
- PostgreSQL
- Авторизация: JWT (подпись запросов)

### Основные требования

- Лаконичная реализация, без избыточных деталей.
- Минимум интерфейса — только API.
- Важно показать чистоту кода и структуру проекта.

### Как клиент API, я должен иметь возможность выполнять следующие действия (через curl)

- Создать пользователя (достаточно только email)
- Проверить баланс конкретного пользователя
- Изменить баланс пользователя (например, пополнить или списать средства)
- Осуществить внутренний перевод средств между двумя пользователями

### Дополнительно

- Примеры 4 curl-запросов для каждого из вышеуказанных действий.
- Остальные детали (валидация, обработка ошибок, структура моделей) — на усмотрение соискателя.
